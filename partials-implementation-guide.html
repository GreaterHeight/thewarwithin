⚔ The War Within — Partials Implementation Guide

This guide tells you exactly what to do in each existing HTML file to switch from copy-pasted nav/footer to the shared JS partial system. Read this fully before touching any file.

1. New Files to Deploy

Copy these four files to your website root (same level as index.html):

  • site.css — all shared CSS (design tokens, nav, footer, drawer, responsive)
  • nav.js — injects top nav + mobile drawer into every page
  • footer.js — injects shared foot-strip into every page (with optional battle band)
  • consent.js — GDPR + NDPR cookie consent banner, deferred analytics loading

2. What to Remove from Every Existing Page

In each HTML file, find and delete the following blocks:

A. The inline <style> block — replace with a link tag

Delete everything between <style> and </style> and replace with:

<link rel="stylesheet" href="/site.css">
⚠ Each page has page-specific CSS inside the <style> block too. Keep those rules. Move them to a second <style> tag immediately after the <link> tag. Only delete the shared sections (design tokens, nav, drawer, footer, responsive for those elements).

B. The GA4 + Clarity scripts in <head>

Delete these two blocks entirely — consent.js will load them conditionally based on user choice:

<!-- DELETE THIS -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-39E56ELNQ5"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-39E56ELNQ5', { ... });
</script>

<!-- AND DELETE THIS -->
<script type="text/javascript">
  (function(c,l,a,r,i,t,y){ ... })(window,document,"clarity","script","x5hzkv6naq");
</script>

C. The nav HTML block

Delete everything from <nav class="b-nav" through </aside> (includes the nav, drawer-overlay div, and mobile-drawer aside).

D. The footer-band nav (battle pages only)

Delete the <nav class="footer-band">...</nav> block on battle pages. It will be rebuilt by footer.js using data attributes (see Section 4 below).

E. The foot-strip footer

Delete <footer class="foot-strip">...</footer>.

F. The nav/drawer JavaScript block

Delete the <script> block that contains the reading bar, nav scroll, drawer open/close, and intersection observer logic. nav.js handles all of this.

3. What to Add to Every Page

In <head> — add these in this order:

<!-- Shared stylesheet -->
<link rel="stylesheet" href="/site.css">

<!-- Consent: must come BEFORE any analytics (replaces GA + Clarity script tags) -->
<script src="/consent.js"></script>

At the TOP of <body> (first line inside <body>):

<!-- Nav placeholder: nav.js replaces this with the full nav + drawer -->
<div id="site-nav"></div>
<script src="/nav.js"></script>
✓ nav.js runs synchronously after the placeholder, so there is no flash of missing nav. The reading bar, scroll behaviour, and drawer all wire up automatically.

At the BOTTOM of <body> (last element before </body>):

<!-- Footer placeholder: footer.js replaces this with the foot-strip -->
<div id="site-footer"></div>
<script src="/footer.js"></script>

4. Battle Pages: Footer Band (Prev/Next Navigation)

For pages that have the battle-to-battle navigation band, pass the data as attributes on the placeholder:

<div id="site-footer"
     data-prev-url="/battle-9.html"
     data-prev-title="When the Spirit Starves"
     data-prev-num="Battle 9"
     data-next-url="/battle-11.html"
     data-next-title="The Inner Saboteur"
     data-next-num="Battle 11"
     data-next-label="Stage IV →">
</div>
<script src="/footer.js"></script>

If any of these attributes are absent, footer.js renders no footer-band (correct behaviour for non-battle pages like FAQ, About, etc.).

6. The Cookie Consent System (GDPR + NDPR)

consent.js handles everything automatically once it is in <head>. Here is what it does:

  • First visit: shows a banner after 800ms with three options — Accept All, Necessary Only, Manage.
  • Accept All: loads GA4 + Clarity immediately, saves consent to localStorage for 180 days.
  • Necessary Only: blocks GA4 and Clarity entirely, saves choice for 180 days.
  • Manage: opens a panel with an Analytics toggle and full NDPA/GDPR legal text.
  • Returning visitors: banner does not show. Analytics loads silently if previously consented.
  • Floating button: a small "Cookie Preferences" button appears bottom-left on all pages, allowing users to change their choice at any time.
  • IP anonymisation: GA4 is configured with anonymize_ip: true (required for GDPR).
✓ The NDPA 2023 and NDPR 2019 require a lawful basis for processing personal data. Consent (opt-in) is the basis used here. The consent record is stored client-side. For a more robust NDPA implementation, you should also update your Privacy Policy to explicitly mention Nigerian data subjects' rights (access, rectification, erasure, portability, objection) and name a Data Protection Officer or contact for Nigerian residents. The privacy.html page should be updated to reflect this.

7. Updated Privacy Policy Required

Your existing privacy.html needs a dedicated Cookies section and an NDPA/GDPR rights section. Add the following content to privacy.html:

## Cookies and Analytics

We use cookies and similar tracking technologies. You can control
your preferences at any time via the Cookie Preferences button
that appears on every page.

**Necessary cookies** are required for the site to function and
cannot be disabled.

**Analytics cookies** (Google Analytics 4, Microsoft Clarity) help
us understand how visitors engage the campaign. These are only
activated with your explicit consent. IP addresses are anonymised.
We do not use analytics data for advertising or sell it to third parties.

## Your Rights (GDPR / NDPA 2023)

If you are in the European Union or Nigeria, you have the right to:
- Access the personal data we hold about you
- Request correction of inaccurate data
- Request erasure of your data
- Object to processing
- Data portability

To exercise these rights, contact: info@techspibus.com
